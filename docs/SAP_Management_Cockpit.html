Data Generators

All Java-based data generators run on sapidess4 via JCo RFC. Each generator has its own directory under /usr/sap/.

Generator Inventory

GeneratorDirectoryProcessKey Transactions
Order to Cash /usr/sap/order_to_cash/ Sales Order → Delivery → Goods Issue → Billing → Payment VA01, VL01N, VL02N, VF01, F-28
Procure to Pay /usr/sap/procure_to_pay/ Purchase Order → Goods Receipt → Invoice → Payment ME21N, MIGO, MIRO, F-53
Plan to Produce /usr/sap/plan_to_produce/ BOM → Routing → Production Order → Confirmation CS01, CA01, CO01, CO11N

CDS Extraction /usr/sap/cds_sql_only/ 8-Phase Pipeline: Dependencies → Metadata → SQL → BFS SE11, DDIC, SE80

Running a Generator

# SSH to the SAP server
ssh root@sapidess4

# Navigate to the generator directory
cd /usr/sap/order_to_cash/

# Compile Java files (if changed)
javac -cp .:sapjco3.jar:gson.jar *.java

# Run the generator
java -cp .:sapjco3.jar:gson.jar -Djava.library.path=. MainClass
All generators require sapjco3.jar and libsapjco3.so in the working directory or on the library path.

Deployment Workflow

Java files are edited locally on Mac and deployed via SCP:

# Edit locally under ~/SAPjco/ or project-specific directory
# Then deploy to production:
scp MyClass.java root@sapidess4:/usr/sap/order_to_cash/

# Compile on server
ssh root@sapidess4 "cd /usr/sap/order_to_cash/ && javac -cp .:sapjco3.jar:gson.jar MyClass.java"
