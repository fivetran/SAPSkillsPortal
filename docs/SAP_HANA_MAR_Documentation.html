Overview

What is MAR?

The Monthly Activity Report (MAR) tool suite is a collection of Python utilities for analyzing SAP HANA table activity. It connects directly to SAP HANA databases to extract metadata, generate SQL queries for tracking INSERT and UPDATE operations by month, and categorize tables by size and data class.

The MAR data helps Fivetran understand:

  • Which tables are actively changing — by counting INSERTs and UPDATEs per month
  • How tables relate to each other — primary keys, reference tables, join fields
  • Table classification — SAP size categories and data classes (APPL0, APPL1, APPL2)
  • Activity trends — monthly volume patterns for capacity planning
GitHub: github.com/fivetran/sap-hana-mar

sql_generator.py

Generates SQL queries that count rows by month using date columns. For each table, it creates a SELECT statement that groups records by year-month based on configurable date fields.

# Example generated SQL:
SELECT
  TO_VARCHAR(ERDAT, 'YYYY-MM') AS MONTH,
  COUNT(*) AS ROW_COUNT
FROM SAPS4H.AUFK
GROUP BY TO_VARCHAR(ERDAT, 'YYYY-MM')
ORDER BY MONTH;

create_table_json.py

Auto-discovers table relationships by querying SAP HANA metadata:

  • Queries DD03L (table field definitions) to extract primary keys
  • Reads CHECKTABLE column to find reference tables automatically
  • Finds common primary key fields between related tables for JOIN conditions
  • Supports explicit join specifications using TABLE:JOIN_TABLE syntax
# Auto-detect reference tables
python create_table_json.py --input raw_table_list.json

# Extract date fields to CSV for configuration
python create_table_json.py --input raw_table_list.json --date-fields-csv date_fields.csv

table_size_and_category.py

Classifies tables using SAP metadata tables:

SAP TableContent
DD09LTable technical settings (size category, data class)
DD07TDomain value descriptions (size category names)
DARTTData class descriptions

Key SAP data classes:

  • APPL0 — Master data (e.g., MARA, KNA1, LFA1)
  • APPL1 — Transaction data (e.g., BKPF, BSEG, EKKO) — most relevant for MAR
  • APPL2 — Organizational data (e.g., T001, T001W)
Setup

Installation

# Clone the repository
git clone https://github.com/fivetran/sap-hana-mar.git
cd sap-hana-mar

# Install dependencies
pip install -r requirements.txt

Dependencies include hdbcli (SAP HANA client for Python), pandas, and standard libraries.

Configuration

# Create config from template
cp hana_config.template.json hana_config.json

Edit hana_config.json:

{
    "host": "sapidess4",
    "port": 30015,
    "user": "SAPHANADB",
    "password": "your_password"
}
Security: hana_config.json is in .gitignore. Never commit credentials. For the SAP HANA password, check the encrypted vault on sapidesecc8 (key: sapidess4_hana).

Workflow

Typical MAR Calculation Flow

  1. Identify tables — Create raw_table_list.json with tables to analyze
  2. Extract metadata — Run create_table_json.py to auto-discover PKs, reference tables, date fields
  3. Classify tables — Run table_size_and_category.py to get size categories and data classes
  4. Configure dates — Review extracted date fields, select the appropriate date column per table
  5. Generate SQL — Run main.py to generate MAR SQL queries
  6. Execute on HANA — Run the generated SQL against HANA to get monthly counts
  7. Analyze results — Review monthly INSERT/UPDATE trends for each table

Quick Start

# 1. Configure HANA connection
cp hana_config.template.json hana_config.json
# Edit hana_config.json with your credentials

# 2. Create table list
cp raw_table_list.template.json raw_table_list.json
# Edit with your tables: ["AUFK", "AFKO:AUFK", "BKPF", ...]

# 3. Extract metadata and generate SQL
python create_table_json.py --input raw_table_list.json
python table_size_and_category.py --json raw_table_list.json
python main.py
